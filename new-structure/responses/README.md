# Customer responses

Solution mapping documents that respond to customer-provided requirements,
architecture asks, or RFPs. Each file pairs the customer's stated needs with the
Microsoft solution proposition, tracking status, gaps and open decisions.

## Conventions

- One file per customer ask, named after the topic
  (for example, [ai-observability-solution-mapping.md](ai-observability-solution-mapping.md)).
- Each file links back to the source asset under [../assets/](../assets/) and to
  any related KDDs in [../decisions/](../decisions/) and OQs in
  [../open-questions/](../open-questions/).
- Status values used throughout response tables:
  - **Proposed** — Microsoft has a candidate solution; pending customer review.
  - **Confirmed** — solution accepted by the customer.
  - **Open** — needs more discovery or a decision; raise an OQ if material.
  - **Gap** — no current Microsoft answer; needs product or engineering input.
